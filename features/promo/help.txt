🚀 智能推广引擎 V3.2 使用手册
━━━━━━━━━━━━━━━━━━━━━━━━━━
本系统基于“素材库-模式库-群组库”三位一体逻辑，实现零阻塞自动化推广。

一、 核心配置流程 (1 -> 2 -> 3 -> 5)
1. 捕获内容 (1xx):
   使用 /tg_save 开启 30 秒锁定模式，直接发送或转发素材给 Bot 即可存入库。
2. 设定模式 (2xx):
   使用 /tg_mode_set 设定触发频率（例如：每积攒 50 条消息发送一次）。
3. 登记群组 (3xx):
   使用 /tg_group_reg 将目标推广群的 ID 绑定到矩阵编号。
4. 启动任务 (5xx):
   使用 /tg_push 关联上述编号，正式开启监听发送任务。

二、 全量指令指南
📂 [数据查询]
/tg_show_lib        - 查看内容库(1xx)、模式库(2xx)、群组库(3xx)全量配置。
/tg_task_list       - 查看当前运行中的 5xx 任务进度、计数及发送统计。

📥 [素材管理]
/tg_save            - 进入 30s 独占捕获态。支持文字、图片、视频、转发消息。
                      (系统自动穿透转发源，锁定原始 MessageID)

⚙️ [逻辑设定]
/tg_mode_set [2xx] inc [数量]
   示例: /tg_mode_set 201 inc 100  (含义: 每收到 100 条消息触发一次)

👥 [矩阵登记]
/tg_group_reg [3xx] [ID1,ID2...]
   示例: /tg_group_reg 301 -100123456,-100654321

🚀 [任务控制]
/tg_push [1xx] [2xx] [3xx]
   示例: /tg_push 101 201 301      (启动联动任务)
/tg_task_del [5xx]
   示例: /tg_task_del 512          (立即停止并销毁指定任务)

三、 技术特性
● [源穿透]: 自动识别频道转发源，即使删除对话缓存素材依然有效。
● [防风控]: 任务执行时自动注入动态时间戳，并带有 2.0s 仿真发送延迟。
● [零阻塞]: 采用 asyncio 异步 Worker，推广任务不影响 Bot 其他功能响应。
